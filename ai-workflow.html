My AI Workflow

01

Architecture Enforcement via AI Rules

I maintain custom rule files that enforce Clean Architecture principles automatically, ensuring proper dependency direction, layer separation, and coding standards.

  • Dependencies always point inward (Presentation → Domain ← Data)
  • Layer-specific constraints for Domain, Data, and Presentation
  • ViewModels follow ObservableObject pattern
  • UI state managed via enums
  • Strict unit testing conventions with XCTest
02

Feature Development with Clean Architecture

AI helps scaffold full features across all layers while maintaining separation of concerns.

  • Domain layer: Models, repository protocols, and use cases
  • Data layer: Repository implementations and DTO mapping
  • Presentation layer: ViewModels and SwiftUI views
  • Unit tests: Full coverage including edge cases and async flows
03

Safe Refactoring Strategy

AI assists in refactoring legacy code incrementally, ensuring stability while improving structure.

  • Preserve existing code during transition
  • Introduce new architecture alongside old code
  • Gradually migrate logic
  • Validate each step before proceeding
04

Unit Test Generation

Automated test scaffolding with comprehensive coverage strategies.

  • Automatic mock and stub creation
  • Structured test cases (one behavior per test)
  • Proper async testing using XCTestExpectation
  • Coverage of success, failure, and edge cases
05

Code Review & Debugging

AI acts as a second reviewer, catching architectural violations and helping debug issues efficiently.

  • Diff-based review before commits
  • Error tracing and root cause analysis
  • Automated PR descriptions using templates
